Start Projects in Seconds

The LaneTerralever Front-side generator is a command-line tool for getting projects started quickly. To get started, make sure you have Node.js installed from the Node.js download page. Then, download Yeoman, the web scaffolding tool, from npm:

npm install -g yo

This will allow you to run the yo command from any of your folders. There are many generators that work with Yeoman, but we created one special for LaneTerralever front-end projects, called generator-lt-frontside. To download it from npm, run

npm install -g generator-lt-frontside

You now have the generator, and can run it anywhere on your computer! Create an empty folder somewhere on your computer, and inside of it run:

yo lt-frontside

And you will have a brand new project, with everything you need to make great front-ends!

Great Built-in Tools

Your project has a few great commands baked into it from our experience working on front-end projects. To start a live-reloading server that compiles your sass on the spot, run the command:

npm start

Now any changes you make in your src directory will show up in your browser. To build the source into a completed version in the dist folder, run:

npm run build

Now all of your files are minified, concatenated, and compiled, ready for any browser to use.

To test your code for style standards and performance metrics, run the test command:

npm test

This will spin perform a Phantomas analysis on the build folder, and produce some nice graphs to check your metrics!

Why use this?

We have been building front-ends for a long time at LaneTerralever, and we've used a lot of technologies, including Bootstrap, LESS, Sass, Angular, Ember, Foundation, Middleman, Grunt, Gulp, Bower, Bundler, npm, CoffeeScript, Jade, Haml, and dozens of other front-end tools. Every project had its own setup, with its own commands and compilation steps. There were no standards, and every project felt like it needed to retread the same water.

With this toolkit, we hope to remove headache of setting up a new project. This toolkit makes it easy to get started quickly on a project and not fuss with any configuration.

We hope to stay up to date with the rotating gambit of new technology, and make it easy to utilize cutting edge technology while still conforming to standards. To do this, we are relying on Bower as a dependency manager to keep our dependencies up to date and handle new versions of technology with one command.

We also want to use new tools that come in to the front-end world, like pre-compiled languages, code conformance checkers, and performance enhancing tools. Grunt allows us to easily incorporate these actions into the easy workflow of testing, building, and working in development.

Technology

Currently, we want to keep this starter template minimal enough for you to use what you want, while using the best set of technology that we believe makes front-end development more reusable and painless. We chose to use Sass, Foundation, and optionally Jade.
